Introduction to Cryptography

Cryptography is a method of securing communication to prevent third parties from being able to discern the meaning. To make messages secure, algorithms are created that are meant to encode a message. Then, for the message to be decoded, an algorithm will have a process which will decode it to get back to the original message.

There are multiple types of cryptography, two of which being Asymmetric key and Symmetric key. Asymmetric key cryptography is often called public key cryptography which is done using two keys. In this case, a person would have two keys which are related to one another: a public key which can be accessed by anyone wishing to send them a message, and a private key that only the person knows. The public key is specific to the recipient and is used by the sender to encrypt the message. When the encrypted message is recieved, the recipient can use their private key (which is related to the public key) to decrypt the message.

A common algorithm for asymmetric key is the RSA algorithm which uses prime numbers, ideally very large prime numbers for the strongest encryption. This will involve the use of modular arithmetic as well.

Symmetric key cryptography uses one key to both encrypt and decrypt messages. The best way to ensure security in symmetric key systems is to protect the secrecy of the key itself. This is the most obvious drawback of this type of system. The most common example of a symmetric key system is a substitution cipher.

The Hill Cipher is an example of a symmetric key algorithm, and it uses matix multiplication and inverse matrices to encrypt and decrypt a message.

When using any sort of algorithm to encrypt a message, the letters must be changed into a numerical value. This has to have a defined method making so that a standard system can be used. For example, the letters corresponding number in the alphabet could be used, but there are also some times where more characters are needed or space values as well. The examples on the RSA Algorithm and Hill Cipher pages use the alphabet values. The ASCII table is another method that is perhaps more standard since it has characters and the values already defined, so the binary or hexadecimal numbers could be used.
